# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via [GitHub Security Advisories](https://github.com/maaoBit/wxl/security/advisories/new).

### What to Include

Please include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution**: Depends on complexity, we'll keep you updated

### Disclosure Policy

- We will acknowledge your report promptly
- We will investigate and fix the issue as quickly as possible
- We will publish a security advisory after the fix is released
- We will credit you for the discovery (unless you prefer to remain anonymous)

## Security Best Practices

When using WXL:

- Keep the application updated to the latest version
- Be cautious when copying sensitive information
- Use the pin feature to protect important clipboard items from auto-cleanup
- Review clipboard history periodically and delete sensitive items

Thank you for helping keep WXL and its users safe!
